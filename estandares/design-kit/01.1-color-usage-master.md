# Color Implementation Master (v5.2)

This guide provides formal patterns for applying the MetodologIA color palette across different technical layers.

## 1. CSS Standard (Variables)
Always use the CSS variables defined in [variables.css](file:///Users/deonto/Documents/Antigravity/site-metodologia/estilos/variables.css).

```css
/* Preferred Usage */
.my-component {
    background-color: var(--bg-surface);
    color: var(--text-primary);
    border: 1px solid var(--border-subtle);
}

/* Glassmorphism Pattern */
.my-card {
    background: var(--bg-elevated);
    backdrop-filter: blur(12px);
    box-shadow: var(--shadow-lg);
}
```

## 2. HTML Standard (Inline/Tailwind)
For critical components or fast prototyping, use the following tokens.

### Inline Styles (Safe)
```html
<div style="background-color: var(--bg-body);">...</div>
```

### Tailwind Classes (v4 Standard)
If using the local build, these tokens are mapped to arbitrary values or custom theme keys:
- `bg-[var(--bg-primary)]`
- `text-[var(--brand-gold)]`
- `shadow-[0_0_20px_var(--brand-gold-glow)]`

## 3. JavaScript Standard (Runtime)
To access colors in dynamic components (e.g., Canvas, Charts, or dynamic styles), use the `ThemeUtils`.

```javascript
/* Standard JS Access */
const getBrandColor = (tokenName) => {
    return getComputedStyle(document.documentElement)
        .getPropertyValue(`--${tokenName}`).trim();
};

const gold = getBrandColor('brand-gold');
```
